## BUG-01
- Краткое описание: Некорректный контракт успешного ответа `POST /api/1/item` (кейсы `TC-01`, `TC-02`, `TC-03`, `TC-04`, `TC-63`).
- Шаги воспроизведения:
  1. Отправить `POST /api/1/item` с валидным JSON-телом.
  2. Проверить структуру ответа.
- Фактический результат: Возвращается только поле `status` со строкой вида `"Сохранили объявление - <uuid>"`.
- Ожидаемый результат: Возвращается JSON-объект объявления с полями `id`, `sellerId`, `name`, `price`, `statistics`, `createdAt`.
- Серьёзность: High.
- Причина серьёзности: Ломается контракт API и становится невозможна корректная проверка/интеграция по спецификации.
- Окружение: `https://qa-internship.avito.com`, Windows 11, Java 21, Maven 3.x, JUnit 5.

## BUG-09
- Краткое описание: Некорректная валидация `sellerID` в `GET /api/1/{sellerID}/item` (кейсы `TC-51`, `TC-53`, `TC-54`).
- Шаги воспроизведения:
  1. Отправить `GET /api/1//item` (пустой `sellerID`).
  2. Отправить `GET /api/1/0/item`.
  3. Отправить `GET /api/1/-1/item`.
  4. Проверить HTTP-статусы.
- Фактический результат: `405` для пустого `sellerID`, `200` для `sellerID=0` и `sellerID=-1`.

- Серьёзность: High.
- Причина серьёзности: Некорректная валидация идентификатора приводит к выдаче невалидных выборок и нарушает предсказуемость API.
- Окружение: `https://qa-internship.avito.com`, Windows 11, Java 21, Maven 3.x, JUnit 5.
